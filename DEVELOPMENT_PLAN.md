# 스마트폰 과의존 개선 웹 프로토타입 개발 계획

## 1. 웹페이지 전체 구조

이 프로젝트는 `index.html`, `style.css`, `script.js` 세 파일로 구성하는 단일 페이지 웹 애플리케이션으로 만든다. 별도의 서버나 빌드 도구 없이 브라우저에서 `index.html`을 직접 열어 실행할 수 있게 한다.

전체 화면은 실제 페이지 이동이 아니라 JavaScript로 화면 섹션을 보여주고 숨기는 방식으로 구성한다.

- 시작 화면: 서비스 소개와 설문 시작 버튼
- 설문조사 화면: 5개 문항을 단계별 카드로 표시
- 습관 추천 결과 화면: 조건문으로 선택된 추천 습관 표시
- 메인 화면: 사용자가 선택한 오늘의 습관과 실천 기록 버튼 표시
- 기록 화면: 최근 7일 실천 현황과 간단한 통계 표시

화면 상단에는 필요한 경우 간단한 제목과 현재 위치를 보여주고, 메인 화면 이후에는 `오늘의 습관`과 `기록 보기`를 오갈 수 있는 작은 내비게이션을 둔다.

## 2. 화면별 주요 기능

### 시작 화면

- 서비스 이름을 보여준다.
- 스마트폰 사용을 강제로 막는 서비스가 아니라 좋은 습관으로 대체하는 서비스라는 설명을 제공한다.
- `나에게 맞는 습관 찾기` 버튼을 누르면 설문조사 화면으로 이동한다.
- 이미 선택한 습관이 LocalStorage에 있으면 `오늘의 습관으로 이동` 버튼도 함께 보여줄 수 있다.

### 설문조사 화면

- 한 번에 한 문항만 보여준다.
- 총 5개 문항을 단계별로 진행한다.
- 진행 바와 `1 / 5` 같은 현재 단계 표시를 제공한다.
- 선택지를 버튼 형태로 보여주어 모바일에서도 쉽게 누를 수 있게 한다.
- 답변을 선택하면 다음 문항으로 이동하거나, 마지막 문항에서는 추천 결과를 계산한다.
- `이전` 버튼을 제공해 직전 문항의 답변을 수정할 수 있게 한다.

### 습관 추천 결과 화면

- 설문 답변을 바탕으로 추천된 습관 하나를 크게 보여준다.
- 추천 습관 이름, 하루 실천 시간, 실천 방법, 추천 이유, 예상 난이도를 표시한다.
- `이 습관으로 시작하기` 버튼을 누르면 선택한 습관을 저장하고 메인 화면으로 이동한다.
- `다른 습관 보기` 버튼을 누르면 같은 답변을 바탕으로 다른 후보 습관을 보여주거나 기본 후보 목록에서 다음 습관을 보여준다.

### 메인 화면

- 사용자가 선택한 습관을 가장 눈에 띄게 표시한다.
- 실천 방법, 예상 소요 시간, 현재 연속 실천 일수, 오늘의 응원 문구를 보여준다.
- `완료했어요`, `일부만 했어요`, `오늘은 못 했어요` 버튼으로 오늘의 실천 상태를 기록한다.
- 같은 날짜의 기록은 새로 추가하지 않고 기존 기록을 수정한다.
- `기록 보기` 버튼으로 기록 화면에 이동할 수 있다.
- 필요하면 `습관 다시 찾기` 버튼으로 설문을 다시 시작할 수 있다.

### 기록 화면

- 최근 7일 날짜를 카드나 작은 원형 아이콘으로 보여준다.
- 완료는 체크 표시, 일부 완료는 세모 표시, 미완료는 엑스 표시, 기록 없음은 빈 원 표시로 나타낸다.
- 연속 실천 일수, 최근 7일 성공 횟수, 전체 성공률을 계산해 보여준다.
- CSS 막대그래프로 최근 7일 성공률을 간단히 표현한다.
- 사용자를 비난하지 않는 짧은 격려 문구를 제공한다.

## 3. 화면 사이의 이동 방식

화면 이동은 `showScreen(screenName)` 같은 함수를 만들어 처리한다. 모든 화면 섹션에는 공통 클래스와 고유 ID를 부여하고, 현재 화면만 `active` 클래스를 갖도록 한다.

기본 흐름은 다음과 같다.

1. 시작 화면에서 `나에게 맞는 습관 찾기` 클릭
2. 설문조사 화면으로 이동
3. 5개 문항에 답변 완료
4. 추천 결과 화면으로 이동
5. `이 습관으로 시작하기` 클릭
6. 메인 화면으로 이동
7. 실천 상태 기록
8. 기록 화면에서 최근 현황 확인

브라우저 새로고침 후에는 LocalStorage에 선택한 습관이 있으면 메인 화면을 기본으로 보여주고, 저장된 습관이 없으면 시작 화면을 보여준다.

## 4. 설문 답변에 따른 습관 추천 조건

추천 방식은 실제 AI가 아니라 조건문 기반으로 단순하게 구현한다. 우선순위가 높은 조건부터 검사하고, 해당 조건이 없으면 관심 활동과 가능한 시간을 기준으로 추천한다.

예상 조건은 다음과 같다.

| 조건 | 추천 습관 |
| --- | --- |
| 자기 전 사용이 많고 수면 개선을 원함 | 자기 전 책 5쪽 읽기 |
| 자기 전 사용이 많고 글쓰기 또는 휴식에 관심 있음 | 오늘 있었던 일을 세 문장으로 기록하기 |
| 스트레스 때문에 사용하고 산책 또는 휴식에 관심 있음 | 저녁 식사 후 10분 산책하기 |
| 스트레스 개선을 원하고 음악에 관심 있음 | 좋아하는 음악 한 곡을 스마트폰 화면을 끄고 듣기 |
| 공부를 피하려고 사용하고 집중력 개선을 원함 | 10분 동안 한 가지 공부에만 집중하기 |
| 학교 또는 공부 시간에 사용이 많고 정리에 관심 있음 | 공부 시작 전 책상 5분 정리하기 |
| 운동 부족 개선을 원함 | 하루 5분 스트레칭하기 |
| 습관적으로 스마트폰을 확인함 | 스마트폰을 확인하고 싶을 때 물 한 잔 마시기 |
| 독서에 관심 있고 사용 가능 시간이 5분 이상임 | 자기 전 책 5쪽 읽기 |
| 운동 또는 산책에 관심 있고 사용 가능 시간이 10분 이상임 | 저녁 식사 후 10분 산책하기 |
| 특별 조건이 맞지 않음 | 하루 5분 스트레칭하기 |

추천 결과에는 `추천 이유`도 함께 만든다. 예를 들어 수면 개선을 선택한 사용자에게는 "자기 전 화면 시간을 줄이고 잠들기 전 마음을 차분하게 만드는 데 도움이 됩니다."처럼 표시한다.

## 5. JavaScript에서 사용할 데이터 구조

JavaScript 데이터는 단순한 객체와 배열로 관리한다.

```js
const questions = [
  {
    id: 'timeOfUse',
    text: '스마트폰을 가장 많이 사용하는 시간대는 언제인가요?',
    options: ['아침', '학교 또는 공부 시간', '저녁', '자기 전']
  }
];
```

```js
const habits = [
  {
    id: 'read_before_sleep',
    name: '자기 전 책 5쪽 읽기',
    duration: '5분',
    method: '잠들기 전 스마트폰을 내려놓고 종이책이나 교재를 5쪽 읽습니다.',
    difficulty: '쉬움',
    tags: ['수면', '독서', '자기 전']
  }
];
```

```js
const appState = {
  answers: {},
  recommendedHabitId: null,
  selectedHabitId: null,
  records: {
    '2026-07-16': 'done'
  }
};
```

실천 상태 값은 다음처럼 단순하게 둔다.

- `done`: 완료
- `partial`: 일부 완료
- `missed`: 미완료

## 6. LocalStorage 저장 방식

LocalStorage에는 하나의 키에 전체 상태를 저장한다.

- 저장 키: `phoneHabitAppState`
- 저장 형식: JSON 문자열

저장할 데이터는 다음과 같다.

```js
{
  answers: {
    timeOfUse: '자기 전',
    reason: '스트레스를 풀기 위해',
    interest: '산책 또는 휴식',
    availableTime: '10분',
    goal: '스트레스'
  },
  recommendedHabitId: 'evening_walk',
  selectedHabitId: 'evening_walk',
  records: {
    '2026-07-14': 'done',
    '2026-07-15': 'partial',
    '2026-07-16': 'done'
  }
}
```

연속 실천 일수는 LocalStorage에 직접 저장할 수도 있지만, 기록 데이터에서 매번 계산하는 방식이 더 안전하다. 기록이 수정되더라도 계산 결과가 자동으로 맞춰지기 때문이다.

LocalStorage 처리 함수는 다음처럼 분리한다.

- `loadState()`: 저장된 JSON을 읽어 기본 상태와 합친다.
- `saveState()`: 현재 상태를 JSON 문자열로 저장한다.
- `resetState()`: 저장 데이터를 삭제하고 시작 화면으로 돌아간다.

## 7. HTML, CSS, JavaScript 파일별 역할

### index.html

- 전체 화면 섹션 구조를 작성한다.
- 시작 화면, 설문 화면, 추천 결과 화면, 메인 화면, 기록 화면의 기본 마크업을 둔다.
- 반복적으로 바뀌는 내용은 JavaScript가 채울 수 있도록 빈 컨테이너를 둔다.
- `style.css`와 `script.js`를 연결한다.

### style.css

- 전체 색상, 글꼴, 여백, 카드, 버튼 스타일을 정의한다.
- 모바일 우선 반응형 디자인을 적용한다.
- 화면 전환 시 사용할 간단한 페이드 또는 슬라이드 효과를 정의한다.
- 진행 바, 기록 아이콘, 간단한 막대그래프 스타일을 작성한다.
- 초록색, 파란색, 크림색 계열을 사용해 밝고 편안한 분위기를 만든다.

### script.js

- 설문 문항 데이터와 습관 후보 데이터를 정의한다.
- 화면 이동, 설문 진행, 추천 계산, 습관 선택, 실천 기록, 통계 계산을 처리한다.
- LocalStorage 저장과 불러오기를 담당한다.
- DOM 요소에 이벤트 리스너를 연결한다.
- 날짜 계산과 최근 7일 표시를 처리한다.

## 11. 예상되는 오류와 해결 방법

### LocalStorage 데이터가 깨진 경우

저장된 JSON을 읽을 때 오류가 발생할 수 있다. `JSON.parse` 실패 시 기본 상태로 되돌리고 새로 저장한다.

### 오늘 기록이 여러 번 추가되는 경우

기록을 배열에 계속 추가하면 같은 날짜 데이터가 중복될 수 있다. 날짜를 키로 사용하는 객체 형태로 저장해 `records[today] = status` 방식으로 수정한다.

### 날짜 계산이 하루씩 어긋나는 경우

`toISOString()`은 UTC 기준이라 로컬 날짜와 다를 수 있다. 사용자의 브라우저 기준 날짜를 `new Date()`에서 연, 월, 일을 직접 조합해 `YYYY-MM-DD` 형식으로 만든다.

### 설문 답변 없이 추천 화면으로 이동하는 경우

마지막 문항까지 답변했는지 확인하고, 누락된 답변이 있으면 해당 문항으로 돌아가게 한다.

### 선택한 습관을 찾지 못하는 경우

저장된 `selectedHabitId`가 현재 습관 목록에 없으면 기본 습관을 보여주거나 시작 화면으로 돌아가게 한다.

### 모바일에서 버튼이 너무 작게 보이는 경우

버튼의 최소 높이를 충분히 크게 잡고, 선택지 버튼은 세로로 배치한다.

### 새로고침 후 화면이 초기화되는 경우

페이지가 로드될 때 `loadState()`를 먼저 실행하고, 선택한 습관이 있으면 메인 화면을 렌더링한다.

## 12. 완성 후 확인해야 할 테스트 항목

- `index.html`을 더블 클릭하거나 브라우저에서 열었을 때 첫 화면이 정상적으로 보이는지 확인한다.
- `나에게 맞는 습관 찾기` 버튼을 누르면 설문 화면으로 이동하는지 확인한다.
- 5개 문항의 선택지가 모두 정상적으로 선택되는지 확인한다.
- 진행 바가 문항 단계에 맞게 변하는지 확인한다.
- `이전` 버튼으로 이전 문항에 돌아가 답변을 바꿀 수 있는지 확인한다.
- 자기 전 사용과 수면 개선을 선택했을 때 수면 관련 습관이 추천되는지 확인한다.
- 스트레스와 산책 관심을 선택했을 때 산책 관련 습관이 추천되는지 확인한다.
- 공부 회피와 집중력 개선을 선택했을 때 집중 관련 습관이 추천되는지 확인한다.
- 추천 결과에서 `이 습관으로 시작하기` 버튼을 누르면 메인 화면으로 이동하는지 확인한다.
- 오늘 실천 상태 버튼을 누르면 화면의 오늘 상태가 바뀌는지 확인한다.
- 같은 날짜에 버튼을 여러 번 눌러도 기록이 하나만 유지되고 상태만 수정되는지 확인한다.
- 기록 화면에서 최근 7일 아이콘이 올바르게 표시되는지 확인한다.
- 연속 실천 일수와 성공률 계산이 실제 기록과 맞는지 확인한다.
- 새로고침 후에도 선택한 습관과 기록이 유지되는지 확인한다.
- LocalStorage를 삭제하면 처음 상태로 돌아가는지 확인한다.
- 모바일 크기 화면에서 글자, 버튼, 카드가 잘 보이는지 확인한다.
- 인터넷 연결이 없어도 모든 기능이 동작하는지 확인한다.
